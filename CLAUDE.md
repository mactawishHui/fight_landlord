## Architecture

### Data flow

```
GameContext (useReducer) → gameReducer (pure) → UI components
                ↑
        AI useEffect (auto-dispatches BID/PLAY/PASS after delay)
```

All game state lives in a single `GameState` object managed by `useReducer` in `GameContext.tsx`. The reducer is pure (`gameEngine.ts`) — it never calls AI directly. AI turns are automated by a `useEffect` in `GameProvider` that fires whenever `state.currentTurn` or `state.phase` changes, scheduling a `setTimeout` (600–1300 ms) then dispatching the AI's action.

### Key files and responsibilities

| File | Role |
|---|---|
| `src/types/index.ts` | All TypeScript types. `Rank` is a numeric enum (3–17); higher = stronger. `PlayerId` is `'human' \| 'ai1' \| 'ai2'`. |
| `src/logic/deck.ts` | `createDeck` (54 cards), `shuffle` (Fisher-Yates), `deal` (17+17+17+3), `sortHand`. |
| `src/logic/combinations.ts` | `detectCombination` — recognises all 14 combo types from an arbitrary card selection using a rank-frequency map. `beats` — comparison logic (rocket > bomb > same-type-same-length-higher-rank). `generateCombinations` — enumerates one representative of each combo shape for AI/hint (not all permutations). `getHint` — weakest valid play. |
| `src/logic/ai.ts` | `aiBidDecision` uses a hand-strength heuristic (joker/2/ace bonuses + bomb bonus). `aiChoosePlay` cooperates with farmers (passes if teammate is leading and has >3 cards), otherwise plays minimum-strength beat. |
| `src/logic/scoring.ts` | `computeScore`: `finalScore = baseScore × multiplier`; landlord win → +2× each farmer, −1× each farmer. |
| `src/logic/gameEngine.ts` | Pure reducer + `createInitialState`. All `DEAL`/`BID`/`PLAY`/`PASS`/`NEXT_ROUND` transitions. All-pass bidding triggers an auto-redeal. |
| `src/state/GameContext.tsx` | `GameProvider` wraps the reducer and fires AI moves via `useEffect`. `useGame()` hook exposes `{ state, playCards, pass, bid, startGame, nextRound }`. |
| `src/constants/cards.ts` | `RANK_LABELS` (maps `Rank` enum → display string, jokers use ASCII `'BJ'`/`'SJ'`), `SUIT_SYMBOLS`, `COMBO_LABELS`. |

### SVG rendering rules — critical constraints

`react-native-svg` has two known pitfalls that have caused crashes in this project:

1. **Rotation must be a number, not a string.** Always use `rotation={180}`, never `rotation="180"`.
2. **CJK (Chinese) characters must not appear inside SVG `<Text>` elements** — they crash on Android devices without CJK fonts installed. All text shown inside SVG must be ASCII or basic Unicode (card suit symbols ♠♥♦♣ are fine). Player names and UI labels must be rendered with React Native `<Text>` outside SVG.
3. **`Dimensions.get('window')` must not be called at module level** — it can return 0 before layout. Use `useWindowDimensions()` hook inside the component instead.
4. **Use `transform="rotate(angle, cx, cy)"` string** on SVG `<G>` elements instead of the `rotation`/`originX`/`originY` shorthand props, which are less reliable.

### Game phases

`idle` → (DEAL) → `bidding` → (all bids complete or bid=3) → `playing` → (hand emptied) → `game_over` → (NEXT_ROUND) → `bidding`

If all three players pass during bidding, the engine immediately redeals.

- **Java**: OpenJDK 17
- **`android/local.properties`** is not committed; recreate with `echo "sdk.dir=/opt/android-sdk" > android/local.properties` before building
